在Ubuntu20.04下安装ArduPilot

发表于 2021-04-20 更新于 2025-03-27 分类于 bolgs ， robotics ， Linux ， 无人机

Python2已经不再更新且被支持了，所以迁移到Python3是迟早的事情。好在ArduPilot顺应潮流，也已经将代码迁移到Python3上了。Ubuntu20.04默认的Python就是Python3，所以在Ubuntu20.04下，安装的过程会有一些不一样。

首先，还是从github克隆ArduPilot：

cd ~
sudo apt install git
git clone https://github.com/ArduPilot/ardupilot.git
cd ardupilot
git submodule init
git submodule update –init –recursive

然后运行ArduPilot自带的依赖安装脚本：

Tools/environment_install/install-prereqs-ubuntu.sh -y
. ~/.profile

接下来检查一下是否安装成功：

cd YOUR_PATH/ardupilot/Tools/autotest
sim_vehicle.py -v ArduCopter –map –console

控制台出现，安装成功！接下来安装mavproxy，使用pip3安装：

pip3 install mavproxy

但是非常遗憾的是，dronekit还不支持python3，比较它最后一次更新是2017年。因此我们用conda新建一个环境安装dronekit。

conda create py27 python=2.7
conda activate py27
pip install dronekit
pip install dronekit-sitl

然后我们测试一个以前试过的脚本：

#####DEPENDENCIES#####

from dronekit import connect, VehicleMode, LocationGlobalRelative, APIException
import time
import socket
import exceptions
import math
import argparse

#####FUNCTIONS#####

def connectMyCopter():

parser = argparse.ArgumentParser(description='commands')
parser.add\_argument('--connect')
args = parser.parse\_args()
connection\_string = args.connect

if not connection_string:
import dronekit_sitl
sitl = dronekit_sitl.start_default()
connection_string = sitl.connection_string()

vehicle = connect(connection\_string, wait\_ready=True)

return vehicle

#####MAIN EXECUTABLE#####

vehicle = connectMyCopter()

连接成功。也就是说，我们在python3环境下运行sitl，在python2环境下运行python脚本。
