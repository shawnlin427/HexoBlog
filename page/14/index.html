Gazebo入门（一）：安装与操作入门

发表于 2021-04-29 更新于 2025-03-27 分类于 robotics ， ROS

本系列基本COPY自Gazebo的官方Tutorial，英文好的可以直接前往Gazebo官网学习，记得自带梯子，不然图片加载不出来。

安装

如果你是Ubuntu用户，只要运行一行：

curl -sSL http://get.gazebosim.org sh

这样安装的是最新版的。或者也可以在应用商店里安装Gazebo9.或者安装ROS Melodic的时候，也会集成Gazebo9.

如果你不是Ubuntu用户，那我建议你把操作系统换成Ubuntu，别费那么多劲去尝试在Centos、Deepin、RedHat之类的系统上装Gazebo了。Gazebo对显卡有一定的要求，目前来看Ubuntu是对Nvidia显卡支持最好的Linux系统。

Gazebo的界面

World界面GUI释义如上。底部的控制条用于控制仿真。鼠标操作如下：

Gazebo操作

Gazebo的界面当然不止这么些，不过接下来我们以练代学。我们会建立一个简单的机器人小车，并给他放上一个深度相机传感器。Gazebo提供了简单的机器人建模工具，但是如果要建立更为复杂的模型，需要编辑SDF文件。

模型编辑页面GUI

在Edit选项栏中选择Model Editor，进入模型编辑界面。

  1. 对模型进行操作的一些工具，很像World界面中的
  2. 左状态栏，也是一些编辑模型的工具，有两个tab：insert，model
  3. 插入页，可以插入基本形状、Custom形状，以及从模型库插入模型，后者包括各种传感器，以及小车、船、无人机等复杂的现有模型。Gazebo支持.dae/.stl/..obj/.svg格式的模型文件。
  4. 模型页，可以设置当前模型的一些参数。一个模型的组成包括Link、Joints、Nested Model（插入的模型）以及Plugin。

Gazebo的模型编辑GUI功能可以覆盖大部分的SDF文件操作，但是不支持以下操作：

  1. 编辑插入的模型，或者把自建模型连接到插入模型上
  2. 插入其他形状的几何体
  3. 对等高图的支持
  4. CAD功能

折腾Linux的一些思考

发表于 2021-04-28 更新于 2025-03-27 分类于 bolgs ， 杂谈 ， thought ， 科研二三事

1. 认清需求

为什么需要用Linux？为了简化操作？为了开发？还是只是因为“Because I can”？

如果是为了开发，需要一个清晰的开发目标。系统和软件的版本非常重要，稳定优先于新特性。不要使用最新版本的系统，老版本有更多的人帮忙试错。

如果是为了简化windows的操作，少操一点心，不要使用Ubuntu。推荐Deepin，就能使用微信（虽然依然频繁跳出），已经远胜于其他系统。

如果只是因为“Because I can”，什么系统都行，但是不要强迫自己用Linux替代Windows。

不要在Ubuntu上做需要大量协作的文档工作，也不要在Deepin上做专业性的开发工作。

2. 及时放弃，学会妥协

这是针对不影响你的需求的问题。比如，对于开发来说，界面的一些外观BUG并不重要。对于日常使用来说，无法安装CUDA也不重要。如果遇到这些问题，网上的解决方案又无法解决，最好的应对方法是：abandon。

比如我在试图用Ubuntu20进行ROS开发学习时，遇到了笔记本无法调节亮度的问题。这个问题困扰了我整整两天，两天内我没有任何产出，也没有学习任何ROS的知识。当我安装了Deepin操作系统后，这个问题就解决了。However, I totally lose the possibility to learn ROS, for Deepin can not support ROS very well. It is not a good bargain, considering that adjusting the brightness through hardware is not that important compared with the compatibility with the ROS of the system I’m using.

3. 什么才是最重要的？

顺畅的开发与学习才是最重要的。

  1. 不要强迫自己改变自己的操作习惯。
  2. 不要为了追求新特性盲目升级软件或系统。
  3. 不要让系统强行做它不擅长的工作。
  4. 布置环境是一件浪费时间和精力却又不得不做的事情，如果你花了一天尝试在新的环境下安装兼容的软件却失败了，你就浪费了一天，没有学到任何东西。
  5. Only your mission matters. Don’t install other application until you really need it. Just work with “it can works”, not “it works the best”. I will now install a Sougou pinyin as I need a Chinese input indeed.

ROS2入门（二）：创建启动文件、环境与包

发表于 2021-04-26 更新于 2025-03-27 分类于 robotics ， ROS

创建启动文件环境和包是标准流程的一部分。启动文件可以让你一行命令启动含有多个node的系统，这样就不用一个一个地启动Node来组成机器人了。

环境是一个包含了指定依赖项的系统运行环境，类似conda中的环境的定义。

包允许你打包你的系统，这样可以分享或很方便地转移系统。

启动创建文件

前面我们使用了小海龟示例，一般我们会运行turtlesim节点和turtle控制节点。后者不是必须的，我们也可以通过直接向topic发布消息的方式控制turtlesim。这里我们创建一个含有两个小海龟的系统，并且让它们两个同步。创建一个python文件，输入：

from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
return LaunchDescription([
Node(
package=’turtlesim’,
namespace=’turtlesim1’,
executable=’turtlesim_node’,
name=’sim’
),
Node(
package=’turtlesim’,
namespace=’turtlesim2’,
executable=’turtlesim_node’,
name=’sim’
),
Node(
package=’turtlesim’,
executable=’mimic’,
name=’mimic’,
remappings=[
(‘/input/pose’, ‘/turtlesim1/turtle1/pose’),
(‘/output/cmd_vel’, ‘/turtlesim2/turtle1/cmd_vel’),
]
)
])

这里我们创造了三个节点，显然前两个是相同的turtlesim节点。这和之前ros2 run <package_name> <node_name>的效果是一样的。第三个节点值得额外的注意，它的executable属性是mimic，也就是模仿的意思。他将/turtlesim1/turtle1/pose这个topic中的所有信息都remap到/turtlesim2/turtle1/cmd_vel这个topic中。因此第二只小海龟会模仿第一只小海龟的动作。保存后，运行：

ros2 launch turtlesim_mimic_launch.py

然后再开启一个命令行，向turtlesim1的/turtlesim1/turtle1/cmd_vel话题发布一条信息：

ros2 topic pub -r 1 /turtlesim1/turtle1/cmd_vel geometry_msgs/msg/Twist “{linear: {x: 2.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: -1.8}}”

信息是这样流动的：我们向cmd_vel发送一条信息，turtlesim1接受后向pose话题发布信息，此时mimic节点订阅了pose话题收取信息，并将该信息发布到turtle2的cmd_vel话题上，turtlesim2从而获得相同的信息。

创建环境

在Linux下，所有的对象都是以文件的方式存储与体现的，因此环境实际上就是包含了一个ROS系统中各种包的路径，直观的体现就是一个文件夹。为什么我们需要一个环境？简单地说，我们不希望一大堆源码和文件把我们的桌面或者/home搞得一团糟，还有我们最不希望发生的就是不同的依赖发生冲突。

mkdir -p ~/dev_ws/src
cd ~/dev_ws/src

这样我们就创造了一个工作空间。我们在这里创造环境。然后我们下载一些包：

git clone https://github.com/ros/ros\_tutorials.git -b foxy-devel

进入ros_tutorials文件夹，下面有四个包：

roscpp_tutorials rospy_tutorials ros_tutorials turtlesim

我们只用到turtlesim这个包。下一步要做的是在这个工作目录下安装依赖项。我们在工作目录的根目录，也就是dev_ws下，执行：

rosdep install -i –from-path src –rosdistro foxy -y

rosdep很有可能还是会接二连三的出现网络问题，没有什么太好的解决办法。下一步是build工作空间，不过在此之前，我们先修改一下turtlesim这个包，这样我们可以确认确实这个包是在当前环境下运行的。修改turtlesim/src下的turtle_frame.cpp，找到第52行的“setWindowTitle(“TurtleSim”);”，改成”MyTurtleSim”然后保存。接下来build这个包：

colcon build

这会创建以下的文件夹：

build install log src

install中包含了当前工作空间的各项设置。下一步要做的就是source这些设置，开启一个新的控制台，这很重要：

cd ~/dev_ws
. install/local_setup.bash

在source的控制台中，运行：

ros2 run turtlesim turtlesim_node

可以看到窗口名称发生了变化。如果要在一个特定环境下运行项目，需要记得source当前环境的设置，也就是. install/local_setup.bash。每次开启一个新的控制台，都需要那么做。

创建包

一个形象的比喻：你可以在ipython中用交互的方式让计算机执行你的想法，但是更常见的还是运行一个py文件。py文件就像ROS里的包。

ROS2支持用C++和Python来写包，作为编程苦手我果断选择了Python。选择Python的话，一个包会包含以下文件：

  • package.xml 描述这个包一些元信息的文件。元信息是”关于信息的信息”。
  • setup.py 描述了如何安装这个包的文件。运行这个py文件就可以安装包了。
  • setup.cfg 这个文件不是必须的，如果包含有可执行项，则需要。ros2 run通过这个文件找到可执行包。
  • /<package_name> 一个目录，含有__init__.py文件。ROS2根据这个找到你的包。

最简单的一个包目录结构如下：

my_package/
setup.py
package.xml
resource/my_package

环境下的包

接下来我们把环境和包结合起来。最佳做法是在环境路径中src文件夹下放所有的包，像这样：

workspace_folder/
src/
package_1/
CMakeLists.txt
package.xml

  package\_2/
      setup.py
      package.xml
      resource/package\_2
  ...
  package\_n/
      CMakeLists.txt
      package.xml

你可能注意到了，基于C++和Python的包可以放在一个环境下。接下来我们实打实地创建一个自己的包。进入src文件夹下：

cd ~/dev_ws/src
ros2 pkg create –build-type ament_python –node-name my_node my_package

--build-type指定了包使用python，–node-name制定了包和其所含节点的名称。这条命令应当会有一下输出：

修改包的元信息

我们知道，包其实就是node的一种表现形式。修改node也是通过修改包来体现的。所以我们该如何修改包呢？首先再来看看我们刚刚创建的包里都有什么：

my_package package.xml resource setup.cfg setup.py test

首先来对package.xml进行修改，这类似于给包打上一下标签，例如它的版本，依赖项，作者的一些信息等。

my\_package 0.0.0 TODO: Package description user TODO: License declaration

<test_depend>ament_copyright</test_depend>
<test_depend>ament_flake8</test_depend>
<test_depend>ament_pep257</test_depend>
<test_depend>python3-pytest</test_depend>

ament\_python

可以修改name、version、description等信息。depend列出了package的一些依赖项，很像maven的管理模式。修改了xml里的信息后，也需要到setup.py里修改：

from setuptools import setup

package_name = ‘my_py_pkg’

setup(
name=package_name,
version=’0.0.0’,
packages=[package_name],
data_files=[
(‘share/ament_index/resource_index/packages’,
[‘resource/‘ + package_name]),
(‘share/‘ + package_name, [‘package.xml’]),
],
install_requires=[‘setuptools’],
zip_safe=True,
maintainer=’TODO’,
maintainer_email=’TODO’,
description=’TODO: Package description’,
license=’TODO: License declaration’,
tests_require=[‘pytest’],
entry_points={
‘console_scripts’: [
‘my_node = my_py_pkg.my_node:main’
],
},
)

package.xml和setup.py里的信息要一一对应。到这里我们大概了解了ROS的文件组织模式，下一章中我们开始尝试设计并运行一个简单的ROS系统。

ROS2入门（一）：ROS中的概念

发表于 2021-04-21 更新于 2025-03-27 分类于 robotics ， ROS

ROS是机器人操作系统（Robot Operating System）的英文缩写。ROS是用于编写机器人软件程序的一种具有高度灵活性的软件架构。但是ROS并不是一个独立的操作系统，而是要依赖于ubuntu等操作系统。ROS2foxy也添加了对Windows等平台的支持，之前ROS只能在Ubuntu操作系统上使用。

ROS的整体布局如上图所示。

ROS NODE（节点）

节点，是一个仅负责单一功能的模块，比如说控制一个轮子，控制一个激光雷达的工作等等。节点之间的通信方式包括topic（话题）、services（服务）、actions（活动）、parameters（参数），后面会一一介绍。节点中还可能会包含一个Subsriber，来接受信息；Pubilsher，用来发送信息。根据功能，也可以分为Service Clinet和Service Server。

一个完整的机器人系统就是由多个节点组成的。可以在Turtlesim中查看节点，来对节点有个更深刻的印象。开启一个新的控制台，输入：

ros2 run turtlesim turtlesim_node

再开一个新的控制台，输入：

ros2 node list

应该能看到：

/turtlesim

此时只有一个节点，turtlesim，就是那只小海龟。下面开启一个新的节点，用来控制小海龟：

ros2 run turtlesim turtle_teleop_key

再次运行ros2 node list，输出：

/turtlesim
/teleop_turtle

Action（活动）：

Action是一个上层的沟通机制，建立在Topic和Services上。它包括三个部分：目标、结果和反馈。

ROS2安装中rosdep update无法进行的问题

发表于 2021-04-20 更新于 2025-03-27 分类于 robotics ， 技术贴 ， ROS

吐槽一下，我能理解长城的意义，但是好歹给科研人员一个官方工具或者官方地点来连接外网吧，一个ROS2安装搞了我两三天，这真的合理吗？

这个问题在外网搜不到，因为外网没有网络问题。即使是使用了科学上网，也会出现无法获取完整内容报错的情况。修改host和ip的方法都宣告无效。最省事的处理办法是使用ROS melodic，并参照https://blog.csdn.net/weixin_39730025/article/details/113348458中的资源，直接拷贝下来。但是也得保证上的去github。

另一种方式就是修改脑残的rosdep机制，一次失败就跳出，也太没有恒心了吧？不过也是，在外网哪有这些烦恼呢……在/usr/lib/python2.7/dist-packages/rosdep2/下的三个文件sources_list.py、gbpdistro_support.py、rep3.py中搜索含有f = urlopen的地方，修改成下面这样：

is_ok=False
count=0
while(not is_ok):
try:
f = urlopen(url_request, timeout=DOWNLOAD_TIMEOUT) #这里是需要网络连接的函数区，url_request在三个文件中命名不一样
is_ok=True
except:
count=count+1 #进行重试
print(‘Retry:’,count)
if count>MAX_RETRY: #超过最大次数MAX_RETRY则跳出
is_ok=True

除此之外，搜索DOWNLOAD_TIMEOUT = 15.0，把数值改高一点，比如30或者60.再声明一下MAX_RETRY的值，比如10.做完之后，rosdep update，然后该干嘛干嘛去吧。

这个方法也不保证成功，只能说减少了敲rosdep update的次数。

当然这个问题可能也只是长城误伤了，因为多试几次还是能够hit到的。但是不管怎么说，都很让人恼火。我也把ROS2的source.list传到了云盘上，希望能帮到人。

要及时记得注销备案主体！

发表于 2021-04-14 更新于 2025-03-27 分类于 bolgs ， 杂谈

我去年在阿里云的服务器到期了，那个时候没啥心思经营网站，所以也没去管它。现在想想，还是挺后悔的，至少要把文章备份下来呀，虽然我也忘了我都写了些啥了。这几天心血来潮，想把自己在腾讯云上的服务器备案一下，却被告知自己因为之前有一个备案，不能正常进行备案了。

去了解了一下备案的相关规则，才发现：

  1. 网站不续费的话，是会被收回的，但是你的备案信息如果没有撤回，又被别人抢注了的话，就好像别人挂着你的招牌做生意。没出什么事儿还好，要是出了什么事儿，可就麻烦了。而且，如果其他人用这个域名搭建的网站和你原来申请的用途不一样，也会被管局找上。
  2. 注销备案的话，还是先联系服务商比较好。不然像我这样，自己在阿里云上操作，把网站注销了，自己就成了空壳主体。需要写一份表格，邮寄到管局请求撤销。

所以说，一定要记得及时注销备案！否则后患无穷。
