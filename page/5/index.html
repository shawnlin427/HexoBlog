如何在ros2_control中切换控制器

发表于 2022-02-06 更新于 2025-03-27 分类于 robotics ， ROS

切换控制器应对这样一种情景：对于伺服器来说，我们有的时候想让他运动到指定位置，有的时候只需要让他以固定速度运动。前者用于自动化控制较多，后者可以用于测试、手动控制等。

假设我们已经加载好了两个控制器，并加载了其中的一个：

ros2 control load_controller forward_position_controller –set-state configure
ros2 control load_controller forward_velocity_controller–set-state configure
ros2 control switch_controllers –start forward_position_controller

lin@lin-G1621-02:~/ros2_control_cranebot$ ros2 control list_controllers
joint_state_broadcaster[joint_state_broadcaster/JointStateBroadcaster] active
forward_position_controller[position_controllers/JointGroupPositionController] active
forward_velocity_controller[velocity_controllers/JointGroupVelocityController] inactive

注意，此时forward_position_controller是active的，说明伺服器由该控制器控制；而forward_velocity_controller是inactive的。同一时刻，一个关节只能由一个控制器控制。

此时使用以下命令切换控制器：

ros2 control switch_controllers –stop forward_position_controller –start forward_velocity_controller

另外需要注意的是，一个关节可以一开始不被任何控制器控制，但是一旦分配了控制器，就不能让它再度不被任何控制其控制。所以如果切换过程中出现错误，可以看看是不是config或者xacro中遗漏了关节。

如何在普通用户权限下访问树莓派的GPIO

发表于 2022-02-05 更新于 2025-03-27 分类于 embedded ， 树莓派

默认情况下要访问树莓派的GPIO，就需要使用root权限。但是有的时候，我们并不想频繁切换成root用户，而且使用ros的时候，系统是无法识别sudo ros2 ….这样的命令的，但是如果不加sudo又无法使用GPIO。以下内容可以让我们在普通用户权限下访问树莓派的GPIO。

https://stackoverflow.com/questions/12133860/accessing-the-gpio-of-a-raspberry-pi-without-sudo

You probably need to be running Raspbian to use RPi.GPIO as a non-root user, possibly it needs to be jessie as well.

Does /dev/gpiomem exist? If so RPi.GPIO should use this device to access the GPIO. If /dev/gpiomem does not exist try:

sudo rpi-update

to make sure you are using a recent kernel, and

sudo apt-get update
sudo apt-get upgrade

to make sure you are using a recent RPi.GPIO.

You will also need to make sure you are a member of the gpio group:

sudo adduser pi gpio

EDITED TO ADD

The Raspbian system appears to be in a state of flux. I’m not sure if it’s a Raspbian or (more likely) a raspberrypi.org problem.

Check that /dev/gpiomem has the correct permissions.

$ ls -l /dev/gpiomem
crw-rw—- 1 root gpio 244, 0 Dec 28 22:51 /dev/gpiomem

If it doesn’t then set the correct permissions as follows

sudo chown root.gpio /dev/gpiomem
sudo chmod g+rw /dev/gpiomem

这两行代码在每次树莓派重启的时候都需要运行。

在搭载Ubuntu MATE的树莓派上安装raspi-config

发表于 2022-02-02 更新于 2025-03-27 分类于 embedded ， 树莓派

raspi-config是一个用于修改树莓派配置的程序，如果树莓派搭载的是默认的Raspi OS，系统默认安装这个程序。但有的时候我们不得不在树莓派上安装Ubuntu，此时又想用raspi-config修改树莓派的配置，就需要手动安装raspi-config了。

首先使用sudo su进入root用户模式，然后将raspberrypi仓库加入源：

echo “deb http://archive.raspberrypi.org/debian/ buster main” >> /etc/apt/sources.list

加入Key：

apt-key adv –keyserver hkp://keyserver.ubuntu.com:80 –recv-keys 7FA3303E

安装：

apt-get update && apt-get install raspi-config

接下来就可以使用raspi-config了，但是由于内核不同，也不保证所有的功能都能使用。

ros2_control_demo代码解析

发表于 2022-01-12 更新于 2025-03-27 分类于 robotics ， ROS

前言

网上有很多ros_control的教程，ros2_control的很少。ros2_control官方出了一个doc，但是看完之后，大部分人也只会照着文档跑demo，对其中各种原理完全不理解。像我一开始就完全搞不清楚，这个ros2_control，到底是怎么怎么通过hw_interface和硬件交互的？ros2_control demo是不涉及这块内容的，因为它要保证没有实际硬件的情况下这个demo也能跑得起来，所以它没有写涉及真实硬件的代码。不过，只要你看完了这篇博文，就会发现，ros2_control真的很简单。

这篇博文通过对ros2_control_demo代码的分析，介绍如何使用ros2_control。

文档结构

一个使用ros2_control的项目不一定要这么设置，只要包含了以下内容：

  • bringup
    • config，里面存放controller的设置，以及其他需要的设置，比如使用joint_tractory_controller的时候可以把轨迹点的信息卸载配置文件里存放。
    • launch，launch file
  • description
    • urdf文件
  • hardware
    • hardware interface，硬件接口，代码的工作量主要在这里
  • test_nodes
    • 一些其他自定义的工具，比如轨迹发布，或者一个自己写的可视化页面等等，随意

总的来说，只有前面三个是必须的，也没有必要像demo项目一样分成几个package分别编译。

rrbot_system_position_only

我们以这个demo为例来探索ros2_control，因为这个demo是最简单的一个。它使用position controller。

看下来我们有几个重点：

  • 默认读取rrbot_controllers.yaml作为配置controllers_file
  • 启动controller_manager，其实不启动也是可以的，不过就要一个一个手动加载了
  • 用controller_manager生成一个joint_state_broadcaster
  • 用controller_manager生成一个也就是forward_congtroller

这里的大部分代码都是可以复用的，除了标为重要的几个部分，涉及到具体的加载文件或路径。
