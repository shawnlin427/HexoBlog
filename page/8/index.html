URDF tutorial（二）：xacro

发表于 2021-12-19 更新于 2025-03-27 分类于 robotics ， ROS

这一章介绍如何在URDF中应用xacro。更详细的介绍访问https://wiki.ros.org/xacro。

xacro也是一种机器人描述语言。URDF要求每一个数据都要手动输入，而xacro允许用参数化的形式输入数据。同时xacro还支持简单的数学运算，以及宏。

使用xacro

一种方法是，编辑好xacro文件后，将其转化为urdf模型：

xacro model.xacro > model.urdf

还有一种方法，就是只保存xacro文件，在launch的时候将其转为urdf：

path_to_urdf = get_package_share_path(‘pr2_description’) / ‘robots’ / ‘pr2.urdf.xacro’
robot_state_publisher_node = launch_ros.actions.Node(
package=’robot_state_publisher’,
executable=’robot_state_publisher’,
parameters=[{
‘robot_description’: ParameterValue(
Command([‘xacro ‘, str(path_to_urdf)]), value_type=str
)
}]
)

这种方法有好有坏，好的是每次修改完文件后不用手动转化了，怀的是launch time会增加。

和其他xml文件一样，需要在开头声明命名空间，xacro才可以正确地转化：

xacro的property

xacro中有两种flag，弈中是xarco:property，另一种是xacro:macro。本节介绍property的使用。

property实际上是声明变量。把下面的urdf转化为xacro：

其中，length和radius的数值是可以用变量来定义的，xacro是这样的：

<xacro:property name=”width” value=”0.2” />
<xacro:property name=”bodylen” value=”0.6” />

因为urdf中无所谓数据的属性（int，str），所以不仅数值可以这样定义，字符串也可以：

<xacro:property name=”robotname” value=”marvin” />

数学运算

xacro可以使用四则运算：

<xacro:property name=”radius” value=”4.3” />

xacro部署了python的math模块，math模块中的运算符和常量都是可以使用的，比如cos、sin、pi、radians等：

<xacro:property name=”R” value=”2” />
<xacro:property name=”alpha” value=”${30/180*pi}” />

宏

宏是xacro的主要目的之一。

先从一个简单的macro入手：

<xacro:macro name=”default_origin”>


<xacro:default_origin />

也就是说macro实际上就是一个block，能够增加代码的复用性。

更复杂的macro

<xacro:macro name=”default_inertial” params=”mass”>




这个macro携带了mass这个参数，并在inertial中使用了这个参数。在使用的时候，就要指定参数的值：

<xacro:default_inertial mass=”10”/>

从这个角度来说，macro也就像编程中的函数。macro不仅能携带参数，还能携带另一个block。此时，参数前面要带一个*，并且在要插入block的地方声明。比如：

<xacro:macro name=”blue_shape” params=”name *shape”>



<xacro:insert_block name=”shape” />





<xacro:insert_block name=”shape” />



<xacro:blue_shape name=”base_link”>

</xacro:blue_shape>

这里声明了一个参数*shape，并在下声明了这里有一块插入。而在使用的时候，在定义了name参数的值后，对于block参数，用xml的语法写在下面。如果有多个block参数的话，定义的时候要按照macro中的声明顺序来。比如：

<xacro:macro name=”pr2_caster” params=”suffix *origin **content **anothercontent”>




<xacro:insert_block name=”origin” />
<xacro:insert_block name=”content” />
<xacro:insert_block name=”anothercontent” />

<xacro:pr2_caster suffix=”front_left”>



0.1








</xacro:pr2_caster>

使用例子

什么时候要使用到xacro呢？我认为主要有两种情况：

  1. 机器人有几个部位是可以复用的，比如轮子；
  2. 机器人的参数比较多，并且互相耦合，同时开发过程中还需要多次修改。

比如下面这个，例子，用xacro避免了左右两条相同的腿要写两次的问题：

<xacro:macro name=”leg” params=”prefix reflect”>














<xacro:default_inertial mass=”10”/>

<joint name="base\_to\_${prefix}\_leg" type="fixed">
    <parent link="base\_link"/>
    <child link="${prefix}\_leg"/>
    <origin xyz="0 ${reflect\*(width+.02)} 0.25" />
</joint>
<!-- A bunch of stuff cut -->


<xacro:leg prefix=”right” reflect=”1” />
<xacro:leg prefix=”left” reflect=”-1” />

有一些比较实用的小技巧：

  1. 用name prefix命名名称相似的物体；
  2. 用math计算joint的origins，这样在改变机器人大小的时候，可以省力；
  3. 使用reflect参数，用来表示在左右两侧对称分布的link，比如轮子、腿。

安装

URDF是机器人描述模型，想要ROS玩的好，URDF少不了。ROS2官方提供了URDF tutorial：

https://github.com/ros/urdf_tutorial/tree/ros2

git clone下来后通过colcon build编译。运行之前，有一些坑需要填上：

  1. 确保你的ros2 foxy版本是2021119以及以后的；

运行第一个示例：ros2 launch urdf_tutorial display.launch.py model:=src/urdf_tutorial/urdf/01-myfirst.urdf

  2. 运行一个节点，发布jointstate和tf
  3. 打开rviz2

URDF语法

URDF使用类似xml的格式，了解sdf的话对URDF肯定不陌生。相比SDF来说URDF更单纯一点，因为只涉及到机器人本身。

URDF也是通过link和joint组成的。一个link的示例如下：

每个link一般会有以下属性：

  • visual：外观，可以用简单的几何体表示，也可以用mesh表示，mesh支持stl和dae。如果使用dae，还能自带材质。
  • material：材质，简单的颜色，还有反光属性。
  • collision：碰撞，一般与外观相同。
  • inertial：惯性，惯性包括质量和转动惯量。质量就用mass表示，而转动惯量用一个3x3的矩阵来表达。由于矩阵是对称的，实际上只要6个元素就能表示转动惯量。ixx/iyy/izz=1e-3 是比较合理的默认值，对应一个边长为 0.1 m 的盒子，质量为 0.6 kg。如果惯性设为0，可能导致机器人模型奔溃。
  • origin：位置，URDF是树形结构，所以子link的位置是相对父link的中心位置而言的

一个joint的示例如下：

一个joint可能会有以下属性：

  • type：类型，一共有continuous、revolute、praismatic、planar、floating，后两种不太常见。前面三种对应连续、旋转、平移。连续关节类似旋转，但是移动距离没有限制，适合用在轮子上。
  • axis：轴，对于旋转关节来说，就是旋转轴，对于平移关节来说，就是平移轴。参照的是绝对坐标系。
  • origin：同link。
  • parent link：相对不动的link
  • child link：相对运动的link
  • limit：对于非continuous的关节，都需要制定effort、lower、upper、velocity。

除此之外，还可以指定粘滞系数、阻尼等，这里不展开。

世界语学习笔记（三）

发表于 2021-12-03 更新于 2025-03-27 分类于 thought ， 世界语

这一节主要学习对句子的分析。

简单句子成分

简单句子可以理解为英文中没有which、when这些修饰的句子，只要没有这些成分，再长的句子都算简单句子。除了最简单的主谓宾结构，句子中还有一下成分：

  • 表语：Ambaŭ Germanio kaj Francio estas demokratiaj ŝtatoj. 俄国和法国都是皿煮国家。民主，demokratiaj，就是表语，表明主语或者宾语的性质。
  • 状语，Alta junulo rapide kuras. 一个年轻人很快的跑，很快的，rapide，就是状语，修饰谓语。
  • 定语，Ŝi estas lernanto, ne tre saĝa, sed tre diligenta. 他是个学生，不是很聪明，但是很勤奋，tre saĝa是形容lernanto的，作定语。

复杂句子成分

复杂句子类似于：I don‘t know where I am、I see what you did last night这种，此时作为主谓宾等句子成分的不是词语或者词组，而是一个句子，此时称为复杂句子成分。在学习世界语的复杂句子成分是，可以和英文进行比对，但是不要彻底收到英文的影响。

  • 从句有指代的情况：类似英文中where、which、who一类的从句，比如：Nur diru al mi, kie vi estas. 只要告诉我你在哪里，kie就是指代哪里，可以发现，此时语序和英文是完全一致的，比如Only if you tell me where you are。
  • 从句无指代的情况：类似I think you are crazy，此时从句开头没有where which who等，在世界语中，需要用一个无实际意义的ke相连表示这种情况，比如Mi aŭdis, ke li iris al Usono. 我听说他去了美国。

以下是一些引导词做疑问句和关系词时的中文含义，但是是不准确的，比如kie也可以用于程度的提问，这里只是做一举例。

世界语学习笔记（二）

发表于 2021-12-03 更新于 2025-03-27 分类于 thought ， 世界语

前言

根据绿网的学习教程，接下来应该是讲复杂语句和分词了，不过直接接触这种复杂的语言规则很可能会劝退学习者，在此之前不如先来学习一些简单的句式，学习完简单的句式后，至少可以根据字典看懂一篇短文章。

陈述句

讲述一个事实，就是最简单的主谓宾结构：

  • La longa plumo estas bona，那个长钢笔是好的。

一般疑问句

中文中，表达疑问的方式就是在陈述句中加个语气词，比如你吃饭了-》你吃饭了吗？不知道是不是受此优美简单的表示方法启发，世界语中，表达一般疑问句的方式就是在句首加个Cŭ：

  • Cŭ vi kuras？你正在跑步吗？

否定句

中文中，表示否定句的方法就是加个否定词，比如我在跑步-》我没在跑步，其他不变。世界语同理，表达否定的方式就是在动词前加个ne：

  • Mi ne kuras，我没在跑步。

特殊疑问句

特殊疑问句和一般疑问句的区别在哪里？区别在于，特殊疑问句，你不能用简单的jes或者ne回答，比如那是什么？你在吃什么？

  • Kio estas tio？那是什么？类似“What is that？”，Kio此时代表疑问主体是主语，一般用在问一个物体的种类时。
    • 特殊一点的，Kiu vi estas，类似是你是谁？而Kio vi estas，类似是你是做什么的？
  • Kie vi estas？你在哪里？类似”where are you“，不过更接近中文的语序。
  • Kiel vi lernas？你学地怎么样？Kiel代表疑问主题是程度，形容动作。
  • Kion vi lernas？你学的是什么？Kion代表疑问主体是做宾语的名词。
  • Kiam vi venis al la lernejo？你什么时候到这个学校来的？Kiam特质对时间的疑问。
  • Kies libroj tio estas？这些是谁的书？Kies代表“谁的”
  • Kia estas via aĝo？你的年龄是多少？Kia代表对一个物体/人的属性的疑问。
  • Kial vi iros al la urbo morgaŭ？为什么你明天要去城里？Kial代表对原因的疑问。

这些疑问句看起来很复杂，但是想一想，英文中when、where、what、who、whose、why、which、how等等不也记下来了吗，而且世界语中的疑问代词也是遵循词性的，例如ki-e就是副词，ki-o就是名词，ki-on就是作为宾格的名词，相对还是好记的。

选择疑问句

选择疑问句，很简单回答就是二选一，不过遵从一般疑问句的形式：

  • Cŭ la floro estas ruĝa aŭ flava？那朵花是红色的还是黄色的？aŭ类似or。

反问句

类似”难道不是吗？“、”是吧？“这种表述。这里有很类似英文，我们知道英文中，对于反问句，只要是肯定的回答就说yes，如果时否定的就说no，世界语也是类似的：

  • Tio estas via plumo, cŭ ne？那是你的钢笔，不是吗？
  • Ne, tio ne estas mia plumo/Jes, tio estas mia plumo，不是，不是我的钢笔/是的，是我的钢笔。

测试

看完前两节，你已经可以在字典的帮助下阅读一般新闻了，而且基本不存在误解。以下新闻来自中国新闻网世界语版：

小节

值得指出的是，世界语的一部分语序还是非常类似中文的，比如在周一和在晚上放在动词前面，但有的时候又像英文那样，把about放在后面。这也是世界语不够统一的地方，但是有人说世界语就是欧洲语，我觉得还是偏颇了。毕竟表意文字就中文一家，俄语、阿拉伯语、韩语都是表音的，日文虽然还保留汉字，平假名片假名也是表音了。也就是说，把世界语的28个字母一一映射到阿拉伯文、俄文甚至彦文，都可以表达。所以作为中文母语者我们应当感到自豪的同时正确看待世界语的地位，而不是因为世界语在表达形式上接近欧洲语言而带有色眼镜去看它。

世界语学习笔记（一）

发表于 2021-12-02 更新于 2025-03-27 分类于 thought ， 世界语

字母

世界语有28个字母，音标如下。

词性

世界语通过词缀来表示词性。

  • 名词：词缀为-o，letero，foto，telefono，lando

  • 动词：词缀为-i，labori，verki，foti，telefoni

  • 形容词：词缀为-a，granda，rapida，bona，sama

  • 副词：词缀为-e，bone，rapide

  • 代词：mi、li他、ŝi她、ili他们、ĝi它、tiu那个

  • 介词：en(in)，sur（on），sub（under），de（of）

  • 数次：从0到10：nul，unu，du，tri，kvar，kvin，ses，sep，ok，naŭ，dek

    • nul就是计算机中的无，0；
    • unu，可以理解成un null，不是无，1；
    • du，double unu，2；
    • tri，类似three，3；
    • kvar，kvin，兄弟俩，罗马数字里IV和V都有V，4，5；
    • ses，sep，姐妹俩，七是seven，6，7
    • ok，和英文的ok一模一样，但是意思是8，好记；
    • naŭ，nine，9；
    • dek，dozen是一打，12个，dek也有组的意思，是10；
    • 其他的数词：cent100，mil1000

  • 连词：kaj（and），sed（but）

  • 冠词：就一个la，类似the

构词

世界语通过构词来拓展表达能力，1984里的新话可能受此启发。

前缀

  • mal-：表示相反的，malbona，malalta（矮的），malbela（丑的）
  • ge-：两性兼备，世界语与西语一样有阴阳性。gepatroj（父母），gefiloj（儿女）
  • re-：再次地，返回地，redoni（返回地拿，返还），reveni（返回的去，回来），revidi（再次地见，重新看到）
  • mis-：错误地，misskribi（错误地写，写错），misuzi（错误地用，误用）

后缀

  • -ist：xxx的人，因为人也是名词，要再加一个o：flegisto（护士）、verkisto（作家）
  • -ar：表示许多同类的东西在一次组成的东西，也要加o：homaro（人类）、vortaro（词典）
  • -ec：形容词转名词，类似英文中的-ness，也要加o：beleco（美）、boneco（好）
  • -ej：xxx物体存在的地方，加o：kokejo（鸡舍）、retejo（网站）
  • -in：表示阴性，加o，也就是在倒数第二个辅音和最末尾的o之间加一个in，knabino（女孩）、patrino（母亲）
  • -em：表示倾向，如果加在动词后，意思类似“喜欢xxx”的，加在形容词后，意思类似“过于xxx的”，由于是形容词，末尾加a：lernema（好学的）、rapidema（仓促的）

复合词

复合词类似中文的组词，飞机就是飞翔flugi+机器maŝino=飞机flugmaŝino。

  • 大部分复合词的特点是会有两个辅音连在一起，比如flugmaŝino，g和m连在一起，以此可以辨认是否是复合词
  • 有些复合词不符合上述规则，比如第二个词首字母是元音的：kokovo，鸡蛋；antaŭurbo，郊区； 还有比如第一个词是介词的，antaŭ，在xxx之前。

词缀不止上面这些，不过这些很常用。其他的可以在熟悉词汇过程中自行归纳。

时态

收到印欧语系影响世界语也是有时态之分的，但是比英文的要简明很多，只需要改变词缀就能表达不同的时态，不用再改变句子结构。

  • -as，类似ing，正在做：Ili kantas，他们正在唱歌。最常用的estas也是esti的正在进行时形式。
  • -is，类似ed，过去完成时：En la matino ili revenis，在早上他们回来了，描述过去的一件事。
  • -os，类似will do，未来时态，Mi revenos en la 12-a de majo，我将回来在五月的12号（我将在5月12日回来）

语气

我们知道英文中有祈使句，虚拟语气，世界语中也有，主要是为了表达强烈的情感或者客气：

  • -u，表示命令或请求，silentu，安静！以u结尾，说出来的情感明显更为强烈。
  • -us，表示虚拟语气，一般和“如果”搭配，Se ŝi estus esperandisto，如果她是世界语者就好了；也可以表达客气，类似英文中有would you和could you的用法，Ĉu vi povus veni morgaŭ?你明天能来吗？

格

格对中国人来说概念不太明确，我们知道有主语和宾语这种东西，但是在中文中一个物体无论是作为主语还是宾语，形式是一样的。而在世界语中，类似英文，物体作为宾语时存在宾格。

  • 世界语的标准语序是主谓宾，也就是mi amas vin，我爱你。不过，有的时候也可以出于强调等原因，调换语序。比如，你的女朋友问你，你爱谁？你可以说，你啊我爱（虽然听起来还是很奇怪），强调的是“你”，对应的是Vin mi amas。又或者说，女朋友说为什么你要给我剥桔子吃？你可以说，爱你啊我，对应的是Amas vin mi。一般情况下，还是按照主谓宾的结构来。
  • -n，在名词后加n，就是宾格。
  • 有的时候会省略主谓语，比如打招呼的时候说saluton，完整的是我向你发去问候，Mi sendas al vi mian saluton。

复数

类似英文中的s，es词缀。

  • -j，是复数的词缀，在名词后加j就可以表达多个的意思，kvin amikoj，五个朋友，j发音类似“yi”。

小结

这一节主要学习的是世界语的字母、发音、构词法、简单的词性变换形式。下一节，将会介绍世界语中几种常用的简单句式。

Prameters用于在运行Node时候指定一些参数，有点类似通过命令行传递参数。如果是通过命令行传递参数的话，可以写成下面这样：

import sys

class JointServer(Node):

def \_\_init\_\_(self):
    super().\_\_init\_\_('joint\_server')
    self.joint = str(sys.args\[0\])
    self.\_action\_server = ActionServer(
        self,
        JointControl,
        self.joint+"\_control\_server",
        execute\_callback=self.execute\_callback,
        cancel\_callback=self.cancel\_callback)
    self.tf\_buffer = Buffer()
    self.tf\_listener = TransformListener(self.tf\_buffer, self)
    self.publisher\_ = self.create\_publisher(Quaternion, '/cranebot/move', 10)

运行的时候，在命令行中传递参数：

python3 xxx.py raise

但是，ROS2不支持通过launch在启动Node的时候附加参数。所以，我们需要使用parameters。

Parameters是什么

Parameters就是一个Node的某个属性，可以在运行前指定，也可以在运行时更改。

什么时候应该用Parameters

当需要在运行Node时指定一些参数，就应该考虑使用parameters。

怎么使用Parameters

下面是一个示例程序片段：

class JointServer(Node):

def \_\_init\_\_(self):
    super().\_\_init\_\_('joint\_server')
    self.declare\_parameter("joint\_type")
    self.joint = self.get\_parameter('joint\_type').get\_parameter\_value().string\_value
    self.\_action\_server = ActionServer(
        self,
        JointControl,
        self.joint+"\_control\_server",
        execute\_callback=self.execute\_callback,
        cancel\_callback=self.cancel\_callback)
    self.tf\_buffer = Buffer()
    self.tf\_listener = TransformListener(self.tf\_buffer, self)
    self.publisher\_ = self.create\_publisher(Quaternion, '/cranebot/move', 10)

首先，需要在初始化节点的时候声明parameter。在声明的时候，也可以直接声明parameter的值，例如：

self.declare_parameter(“joint_type”, “raise”)

这样在不加参数运行的时候，parameter会使用其默认值。然后，使用以下命令获取参数内容：

self.joint = self.get_parameter(‘joint_type’).get_parameter_value().string_value

根据parameter的属性不同，获取的可以是bool_value、integer_value、double_value等。

运行的时候指定Parameter

运行格式为：ros2 run [PackageName] [NodeName] –ros-args -p [ParamName]:=[ParamValue]

ros2 run cranebot joint_server –ros-args -p joint_type:=raise

远程Git仓库的使用：以Gitee为例

发表于 2021-11-20 更新于 2025-03-27 分类于 技术贴

远程Git仓库本质上说是部署在远程服务器上的Git仓库，Gitee、Github这种公开的远程服务使用的比较多。

创建仓库、分支等

创建仓库一般在服务网站上就可以完成，例如创建仓库：

以及创建分支：

将远程仓库保存到本地

方法1：

先将仓库clone到本地，修改后再push到 Gitee 的仓库

$ git clone https://gitee.com/用户个性地址/HelloGitee.git #将远程仓库克隆到本地

在克隆过程中，如果仓库是一个私有仓库，将会要求用户输入 Gitee 的账号和密码。按照提示输入即可。

当然，用户也可以通过配置本地的git配置信息，执行git config命令预先配置好相关的用户信息，配置执行如下：

$ git config –global user.name “你的名字或昵称”
$ git config –global user.email “你的邮箱”

在 Gitee 平台中，强烈建议您在【设置-多邮箱管理】中的“提交邮箱”与上面配置信息中的邮箱地址保持一致，这样平台就能及时地统计您在平台中提交代码的贡献了。

修改代码后，在仓库目录下执行下面命令

$ git add . #将当前目录所有文件添加到git暂存区
$ git commit -m “my first commit” #提交并备注提交信息
$ git push origin master #将本地提交推送到远程仓库

这种方式一般用于，在copy其他人的代码，或者在新设备上部署代码的时候，也就是从无到有的过程。

方法2：

本地初始化一个仓库，设置远程仓库地址后再做push

和方法1的差别，在于先创建仓库。

$ git init
$ git remote add origin https://gitee.com/用户个性地址/HelloGitee.git

这样就完成了版本的一次初始化。
接下去，进入你已经初始化好的或者克隆仓库的目录,然后执行：

$ git pull origin master

修改/添加文件，否则与原文件相比就没有变动。

$ git add .
$ git commit -m “第一次提交”
$ git push origin master

然后如果需要账号密码的话就输入账号密码，这样就完成了一次提交。此时，你可以在你的个人面板、仓库主页查看到你的提交记录。

这种方法一般用于本地已经有了代码，这时候想把它提交到gitee上的时候。

提交冲突

在新建仓库时，如果在 Gitee 平台仓库上已经存在 readme 或其他文件，在提交时可能会存在冲突，这时用户需要选择的是保留线上的文件或者舍弃线上的文件，如果您舍弃线上的文件，则在推送时选择强制推送，强制推送需要执行下面的命令(默认不推荐该行为)：

$ git push origin master -f

如果您选择保留线上的 readme 文件,则需要先执行：

$ git pull origin master

创建、切换分支

方法1

第一种方法是，在远程仓库上创建分支后，本地就可以直接和远程仓库同步。然后就可以直接使用git checkout进行分分支的切换。

方法2

第二种方法是，在本地先创建分支：

git checkout -b testbranch

然后推送到远程仓库：

git push origin testbranch

这样也可以同步。

总结

总结而言，创建仓库和分支都有两种方法，一种是在gitee或者github上创建后，在本地同步；还有一种是在本地创建后，推送到远程仓库。

图像处理踩坑：Uint8

发表于 2021-11-18 更新于 2025-03-27 分类于 tech ， python

这两天在做图像处理作业，其中涉及到计算两个像素点之间的距离，但是怎么算都算不对。后来发现，是数据格式的锅。

uint8一般用来储存图像，值在0到255之间，因此如果让uint8格式的1-2，得到的结果会是254.为了避免这一点，需要在计算之前把数据格式转为int或者其他格式，比如这样：

# 距离函数
def distance(pix1_lab, pix1_loc, pix2_lab, pix2_loc, m, s):
dlab = norm(array(pix1_lab, dtype=’int64’)-array(pix2_lab, dtype=’int64’))
dxy = norm(pix1_loc-pix2_loc)
dist = dlab**2 + (m/s*dxy)**2
#print(dlab, dxy, dist)
return dist
